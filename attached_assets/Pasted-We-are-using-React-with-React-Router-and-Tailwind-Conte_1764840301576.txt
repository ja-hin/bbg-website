Context:

- Device Type (select: Mobile / Laptop)

- Find Plans button

On another section / page we already have 4 plan cards designed (do NOT change their visual design or copy):
- Laptop BBG
- Mobile BBG
- Laptop Extend+
- Mobile Extend+

Requirement:
When the user fills the form and clicks “Find Plans”:

1) Navigation
- Redirect the user to the URL `/plans`.
- Pass along:
  - selected device type (mobile or laptop)
  - device purchase date
  - (brand can be ignored for now in the logic, just pass it if it is already part of the form state)

- Compute whether the device purchase date is within 6 months from “today”.
  - Use JavaScript Date objects.

    - Create a `cutoffDate` by subtracting 6 months from today.

3) Plan visibility rules
On `/plans`, show only the plans that are applicable based on:
- Device Type
- Whether the device is ≤ 6 months old or > 6 months old

Exact rules:

     - Show: Mobile BBG card AND Mobile Extend+ card

B) If device purchase date is more than 6 months from today:
   - If device type is Mobile:

     - Hide: Mobile BBG, Laptop BBG, Laptop Extend+

     - Show: ONLY Laptop Extend+ card

Implementation hints:

Output:
